# 求职助手（Job Hunt Copilot）

把求职里"高价值但耗时"的环节自动化：简历诊断 → 定位 → 真实岗位评估 → 定制材料 → 招聘方应答。目标：用更精准的投递换更多面试，而非无脑海投。

## 候选人配置（可编辑——换人只改这块）
```
姓名: 朱文璇 (Jules.Z)
地点/形态: 上海 · 全职 · 上市地不限
薪资: 税前年包 ≥ 25w，越高越好，更看重发展空间
经验档: 约 2.5 年全职(美股上市公司 IR) + 券商研究/机构销售/审计实习
定位: IR-track(投资者关系/证券事务/董秘办/投后/募资)，非投资 deal 岗
赛道优先级: 1)医疗/生物医药(港股18A) 2)半导体/硬科技 3)智能/机器人/汽车 4)一级市场PE/VC 5)新能源
真实强项: IR执行、双语、合规纪律、叙事翻译；独立顶级研究能力有工具放大成分，据此偏执行+叙事+协调类岗位
```

## 硬边界（不可越）
- 不代填账号密码、不代登录。
- 不写脚本抓取 Boss/猎聘/领英(违规+反爬封号)；真实岗位经 Claude in Chrome 在已登录会话读取；详情页反爬会卡死，不硬刷。
- 投递/发消息/提交表单：内容先给用户看，本人确认后再发；绝不自动群发。

## 五阶段工作流
1. 简历诊断 → `references/resume-analysis.md`（双视角、诚实短板）。
2. 定位匹配 → `references/positioning-match.md`（实时行研）。
3. 真实岗位评估 → `references/job-evaluation.md` + `references/browser-workflow.md`（双维度打分 + 硬性预筛）。
4. 简历/打招呼语定制 → `references/application-tailoring.md`（默认开场白走量，重点岗微调）。
5. 招聘方应答 → `references/recruiter-response.md`（HR 回复后精修，最高 ROI）。

## 硬性预筛规则（阶段3 自动执行，详见 job-evaluation.md）
排除：经验>候选人档(5-10年/10年以上)、JD 明确要求不具备的硬条件(FA/PE/VC背景/美元基金资源/保代)、募资岗要求"必备募资经历/自带LP资源"、已联系过的公司、纯 deal 投资岗。
募资岗细则：基金 GP 侧、要求"募资 track record / 自带 LP 资源 / 政府募资关系"的岗位一律排除(候选人无 LP 资源)；公司侧投融资岗(实业公司为自己融资)保留。一级市场优先用「投后管理 + IR」切入,不投纯募资。

## 关键词配方
好用：投资者关系/IR/证券事务代表/董事会秘书/投后管理/募资/投融资/ECM。避免单搜：资本市场(律师)、战略投资(deal岗)。

## 网页看板（web/）
`web/index.html` 是数据驱动的求职指挥台：按行业分区、可按 行业/岗位/状态 筛选、漏斗统计、状态追踪(待投→已打招呼→已回复→面试→关闭，存 localStorage)、一键复制打招呼语。岗位数据在 `web/jobs.js`(`window.JOBS_DATA`)——每轮把新筛+打分好的岗位写进去即可。可用 GitHub Pages 托管成固定网址(见 README)。

## 每日循环（稳态）
1. 每早 Chrome 定时任务自动收集新岗位(prompt 见 browser-workflow.md)。
2. Claude 打分 + 预筛 → 更新 `web/jobs.js`。
3. 用户在看板投一批、点状态、复制进展。
4. HR 回复 → 精修应答(阶段5)。
放大器：猎聘挂简历+开放猎头、LinkedIn(外资/港股)、目标公司官网/内推。

## 借鉴（同类工具）
看板状态流、漏斗统计、跟进提醒、持久去重、逐岗定制——借思路；不抄自动群发(反爬+封号)。
